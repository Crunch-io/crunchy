Table ðŸ”—

All datasets contain a /table/ endpoint which allows access the full data values. It provides granular control over the rows and columns for each dataset.

Fetching values ðŸ”—

GET ðŸ”—

Dataset editors can GET to this resource and obtain a Shoji Table of the datasetâ€™s data. It will expose all the variables that are visible by the authenticated user (Public + personals created by them if requested) as well as the exclusion filter applied (if any).

To include the personal variables on the output table the client should include the include_personal GET parameter on the request with a True value.

A metadata section contains the definitions of all the variables matched by variable ID with the corresponding entry under data.

Dataset viewers can only access the metadata portion of the response. This means they cannot make use of the limit and offset parameters to query data unless the datasetâ€™s setting viewers_can_export is set to True, else the server will respond with a 403 response.
